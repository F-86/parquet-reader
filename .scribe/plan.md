# AGENTS.md 重构计划

## Brief

- 文档类型：`AGENTS.md`
- 目标受众：AI agent / 编程助手
- 核心目标：让 agent 在本项目中做 Rust Parquet TUI 开发时，快速知道项目定位、常用命令、架构边界、必须遵守的实现规则和禁止行为。
- 必须保留：
  - Python 原型只能作为行为参考。
  - 数据读取默认分页，禁止默认全量加载。
  - TUI、状态、数据访问、格式化需要分层。
  - 筛选表达式必须明确语义边界。
  - 新增依赖需有理由。
  - 失败路径、空数据路径、大文件、宽表、复杂类型是一等场景。
  - 不提交真实数据、大文件、临时文件、target 产物。
  - 不写 `ls` / `grep` 可得的信息。
- 可删减：
  - 重复表达。
  - 过于抽象、不能直接执行的句子。
- 语气风格：短句、命令式、规则优先。
- 目标语言：中文。

## Outline

1. `# parquet-reader`
   - 用项目名作为标题，而不是文件名。
2. `## 项目定位`
   - 简述 Rust Parquet TUI Viewer，以及 Python 原型的参考边界。
3. `## 常用命令`
   - 列出 agent 高频验证命令：`cargo fmt`、`cargo check`、`cargo test`。
4. `## 架构边界`
   - 规定 TUI、App State、Data Access、Formatting 的职责边界。
5. `## 数据读取规则`
   - 规定分页、过滤、count、大文件和异常 Parquet 的保守策略。
6. `## TUI 交互规则`
   - 保持 vim/k9s 风格快捷键，新增交互不得破坏既有习惯。
7. `## 值显示与错误处理`
   - 规定 NULL、宽字符、二进制、嵌套结构、超长单元格、错误信息的处理方式。
8. `## 依赖与实现约束`
   - 新增依赖要说明理由，避免无关重量级依赖，保持跨平台终端兼容。
9. `## 文档维护规则`
   - 文档、需求、实现同步；不要写可直接检索得到的事实。
10. `## 禁止行为`
    - 集中列出不得提交数据、不得全量加载、不得混层、不得提交生成物等硬性禁止项。

## 特殊注意事项

- 不把 `AGENTS.md` 写成 README；它只写给 AI agent。
- 不列当前目录树、Cargo 依赖版本、文件清单等可检索事实。
- 命令只写高频验证命令，不写长教程。
- 当前文件是已有文档；写入前需要用户确认是否直接覆盖，或是否要保留备份。

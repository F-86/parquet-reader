# 09. 里程碑、风险与待决策项

## M1：最小可启动查看器

交付能力：

- CLI 接收可选文件路径。
- 不带文件路径时进入 TUI 空状态。
- TUI 左侧显示贯穿全高的文件列表侧边栏，根目录为程序运行目录。
- TUI 中按 `d` 聚焦左侧文件列表。
- 右侧顶部 tab 栏显示当前文件或 `[No file]`。
- 不常驻显示 key hints，按 `h` 打开帮助弹窗。
- 读取 Schema。
- 显示第一页数据。
- 支持退出。

验收标准：

- 有效 Parquet 文件能通过 CLI 直接进入 TUI。
- 不带文件启动时能进入空状态。
- 空状态可以通过左侧文件列表选择本地 `.parquet` 文件。
- 打开文件后右侧顶部 tab 栏显示当前文件名。
- 左侧文件侧边栏不被底部 key hints 或其他区域挤压。
- 按 `h` 能打开帮助弹窗，`Esc` 或 `h` 能关闭。
- 无效路径能给出明确错误。
- 退出后终端恢复正常。

## M2：导航与分页

交付能力：

- 行导航。
- 水平滚动。
- 上一页和下一页。
- 状态栏显示行范围、页码和列数。

验收标准：

- 翻页不改变筛选条件。
- 上一页不会越过第一页。
- 最后一页不会越界。

对应文档：

- [`03-app-state-and-actions.md`](./03-app-state-and-actions.md)
- [`06-tui-layout-and-navigation.md`](./06-tui-layout-and-navigation.md)

## M3：Schema 与筛选

交付能力：

- Schema 视图。
- 筛选输入弹窗。
- 筛选条件显示在底部 status 栏。
- 筛选重置。
- 查询错误展示。

验收标准：

- 切换 Schema 不破坏数据视图状态。
- 提交筛选后 offset 归零。
- 提交筛选后 status 栏显示当前筛选条件。
- 非法筛选表达式不导致崩溃。

对应文档：

- [`07-filtering-and-schema-view.md`](./07-filtering-and-schema-view.md)

## M4：健壮性与体验

交付能力：

- 类型显示策略完善。
- 宽字符和截断处理。
- 更完整测试。
- 终端恢复和错误路径打磨。

验收标准：

- 复杂类型和异常值不会破坏表格布局。
- 空数据和错误路径有稳定展示。
- 核心状态转移有测试覆盖。

## 当前推荐技术栈

| 领域 | 当前推荐 | 状态 | 说明 |
|------|----------|------|------|
| CLI | `clap` | 建议采用 | 面向用户的 CLI 需要稳定 help 和错误提示 |
| 错误类型 | `thiserror` | 建议采用 | 保持跨层错误结构化 |
| TUI | `ratatui` + `crossterm` | 建议采用 | 兼顾布局能力、事件控制和跨平台终端支持 |
| Parquet 查询 | `datafusion` | 建议先评估并默认采用 | 纯 Rust 查询路线，支持 SQL/filter 和 Parquet 扫描 |
| 值宽度 | `unicode-width` | 建议采用 | 避免按字节截断导致终端错位 |
| CLI 测试 | `assert_cmd` | 实现 CLI 后采用 | 用于断言命令退出码和输出 |
| 临时文件 | `tempfile` | 需要生成样例文件时采用 | 自动清理临时目录和文件 |
| 快照测试 | `insta` | 可选 | 等 TUI 输出稳定后再引入 |

## 技术选型变更规则

- 改变推荐库前，先更新对应编号设计文档。
- 如果变更影响架构边界，例如从 `datafusion` 换成 `duckdb`，需要同步更新 `04-data-access.md` 和本文档。
- 如果变更只是测试辅助库，不需要 ADR；如果变更影响分发、查询语义或用户输入语法，应新增 ADR。

## 风险与缓解

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| Count 代价过高 | 打开或筛选后卡顿 | 懒统计、缓存、后台任务、允许显示未知 |
| 查询引擎依赖过重 | 构建慢、分发复杂 | 用 Data Access trait 隔离，先做小样例评估 |
| 复杂类型显示不可控 | 表格布局破坏 | 统一 Formatting Layer，先摘要再详情 |
| 宽字符截断错误 | UI 错位 | 使用显示宽度计算，不按字节截断 |
| 筛选表达式语义不清 | 用户误解能力或安全边界 | UI 和文档明确表达式来源与限制 |
| 查询阻塞 UI | TUI 无响应 | 后续引入异步查询、取消任务或进度状态 |
| 终端恢复失败 | 用户 shell 状态异常 | 统一终端生命周期管理，错误路径也执行恢复 |
| 文件侧边栏状态复杂 | 空状态、已有文件、焦点切换、目录切换和选择文件之间状态混乱 | 侧边栏动作进入 App State 状态机，并为焦点切换、目录边界和切换文件写单元测试 |
| 多 tab 状态膨胀 | 每个文件都有分页、筛选和滚动状态，容易互相污染 | 第一阶段只实现单 tab 展示，多 tab 后续再扩展，并让每个 tab 保存独立状态 |
| 帮助信息挤占布局 | 常驻 key hints 会压缩文件侧边栏和数据表 | 不常驻显示 key hints；按 `h` 使用 btop 风格弹窗展示帮助 |

## 待决策项

- 第一阶段数据访问路线：当前建议先采用 `datafusion`；若构建、分发或异步封装成本不可接受，再评估 Arrow 原生或 DuckDB 绑定。
- 是否默认计算精确总行数。
- 筛选表达式是直接暴露查询引擎语法，还是设计受限 DSL。
- 是否需要异步查询与取消。
- 左侧文件侧边栏是否需要模糊搜索、最近文件、目录书签、隐藏非 Parquet 文件等增强能力。
- Tab 栏是否支持多文件同时打开、切换、关闭以及新文件打开策略。
- 帮助弹窗是否需要搜索、分页或按上下文只显示当前模式快捷键。
- 是否支持列选择、排序、单元格详情弹窗、导出当前页。
- 是否需要为大型样例建立单独的性能测试集。

## 决策记录规则

- 做出技术选型后，在本文档中更新待决策项状态。
- 影响较大的单个决策应新增 ADR，而不是把完整争论写进本文档。
- 已延期能力应写明延期原因和预计进入的里程碑。

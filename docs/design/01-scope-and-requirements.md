# 01. 范围与用户体验需求

## 目标

- 单文件 Parquet 查看。
- 支持启动时直接传入文件路径，也支持先进入 TUI 后按 `d` 选择 Parquet 文件。
- 顶部显示 Neovim 风格 tab 栏，用于展示当前打开的 Parquet 文件；后续可扩展为多文件 tab。
- 左侧显示 LazyVim 风格文件列表侧边栏，根目录为程序运行目录，用于浏览和选择本地 Parquet 文件；侧边栏从屏幕顶部贯穿到底部，不被 tab 栏、状态栏或帮助区域挤压。
- 终端 TUI 交互。
- 默认分页读取，不全量加载数据行。
- 数据视图显示列名和当前页行数据。
- Schema 视图显示字段序号、字段名和类型信息。
- 支持键盘导航、水平滚动、翻页、筛选输入、筛选重置和退出。
- 状态栏显示视图、行范围、页码、列数、筛选状态和错误提示；当前文件名优先显示在顶部 tab 栏。
- 将 TUI、应用状态、数据访问和值格式化分层，便于后续替换底层查询实现。

## 非目标

- 不编辑 Parquet 文件。
- 不修改源文件。
- 不默认导出当前页或筛选结果。
- 第一阶段不支持多文件 join、glob、远程对象存储路径。
- 第一阶段 tab 栏只要求显示当前文件；多文件同时打开、切换和关闭是后续能力。
- 第一阶段的 TUI 内文件侧边栏只用于浏览程序运行目录下的本地文件并选择单个 Parquet 文件，不做批量目录管理。
- 第一阶段不提供完整 SQL 工作台体验。
- 第一阶段不承诺复杂筛选表达式是安全沙箱。
- 不为了显示总行数而默认扫描全部数据。

## CLI 启动需求

- 用户可以通过命令行传入 Parquet 文件路径并直接打开。
- 用户也可以不传入文件路径，直接进入 TUI 空状态。
- 启动时传入的文件不存在、不可读或不是有效 Parquet 文件时，给出清晰错误。
- 未传入文件时，TUI 显示空状态、左侧文件列表和操作提示，例如“按 `d` 聚焦文件列表”。
- 初始化成功后进入 TUI 数据视图。

## 数据视图需求

- 默认显示数据表。
- 表头显示列名。
- 当前页显示固定数量的数据行。
- 单元格值转换为终端友好的文本。
- 超长值截断显示，并用省略标记表示仍有剩余内容。
- NULL 值使用稳定文本表示。
- 列表、Map、Struct、二进制等复杂值必须有可预测的降级显示策略。

## Schema 视图需求

- 用户可以在数据视图和 Schema 视图之间切换。
- Schema 视图至少显示字段序号、字段名、逻辑类型或物理类型。
- Schema 视图不参与数据分页和筛选。

## 筛选需求

- 用户可以按 `/` 打开筛选输入弹窗。
- 在筛选弹窗中提交筛选后关闭弹窗、刷新结果集，并将分页位置重置到第一页。
- 筛选生效后，下方 status 栏必须显示当前筛选条件。
- 用户可以重置筛选。
- 筛选失败不应导致程序崩溃；应保留当前界面并显示错误。
- 如果筛选表达式透传给底层查询引擎，UI 和文档必须明确说明表达式语义。

## 导航与快捷键

| 快捷键 | 行为 |
|--------|------|
| `j` / `↓` | 下移一行 |
| `k` / `↑` | 上移一行 |
| `h` | 打开帮助弹窗 |
| `←` | 向左滚动 |
| `l` / `→` | 向右滚动 |
| `H` | 滚动到最左列 |
| `L` | 滚动到最右列 |
| `J` | 移动到当前页底部 |
| `K` | 移动到当前页顶部 |
| `n` / `PageDown` | 下一页 |
| `p` / `PageUp` | 上一页 |
| `/` | 打开筛选输入弹窗 |
| `d` | 聚焦/切换左侧文件列表 |
| `r` | 重置筛选 |
| `s` | 切换 Schema 视图 |
| `Esc` | 取消当前输入或弹层 |
| `q` | 退出 |





## 筛选弹窗需求

筛选输入不占用常规布局空间，按 `/` 时以弹窗形式显示。

- `/` 打开筛选弹窗。
- 弹窗参考命令输入框样式，居中或靠上覆盖右侧内容区。
- 弹窗打开时，背景界面保持不变。
- `Enter` 提交筛选并关闭弹窗。
- `Esc` 取消筛选输入并关闭弹窗，不改变当前筛选条件。
- 弹窗中应展示当前筛选条件，便于编辑。
- 筛选错误关闭弹窗后显示在状态栏或错误消息区域，不应破坏当前数据视图。
- 筛选成功后，status 栏显示完整或截断后的筛选表达式，例如 `filter: col > 10`。

## 帮助弹窗需求

TUI 不常驻展示 key hints，避免挤占数据表和左侧文件列表空间。

- 用户按 `h` 打开帮助弹窗。
- 帮助弹窗参考 btop 的帮助界面：居中覆盖、带边框、分组展示快捷键。
- 弹窗内容至少包含：全局、文件侧边栏、表格导航、筛选、Schema、tab 相关快捷键。
- 弹窗打开时，主界面保持在背景中，不改变当前状态。
- `Esc` 或再次按 `h` 关闭帮助弹窗。
- 帮助弹窗不触发数据查询。

## 顶部 Tab 栏需求

目标交互参考 Neovim 的 tabline：屏幕最上方显示当前打开的 Parquet 文件。

第一阶段要求：

- TUI 顶部保留一行 tab 栏。
- 未打开文件时显示空 tab 或提示，例如 `[No file]`。
- 打开文件后显示文件名，路径较长时只显示 basename 或截断后的路径。
- 当前 tab 需要有明显高亮。
- tab 栏只负责展示当前文件，不承载复杂操作。

后续扩展：

- 支持同时打开多个 Parquet 文件。
- 支持在 tab 之间切换。
- 支持关闭当前 tab。
- 支持按 `d` 选择文件时选择“替换当前 tab”或“打开为新 tab”。

## 文件侧边栏需求

- TUI 左侧固定显示文件列表侧边栏。
- 侧边栏从屏幕顶部贯穿到底部，不被顶部 tab 栏、状态栏或弹窗预留区域挤压。
- 侧边栏根目录是启动程序时的当前工作目录。
- `d` 用于聚焦或切换到文件列表侧边栏。
- 第一阶段只需要浏览本地目录并选择单个 `.parquet` 文件。
- 侧边栏应显示目录项和 `.parquet` 文件；其他文件可以隐藏或弱化显示。
- 支持进入子目录和返回上级目录，但不能越过根目录。
- 选择文件后，应用读取 Schema 和第一页数据，并进入数据视图。
- 取消侧边栏焦点后回到主区域，不改变当前已打开文件。
- 选择新文件后，应清空旧文件的筛选条件、分页位置、选中行和错误状态。
- 文件不存在、不可读或读取失败时，保留 TUI 并展示可恢复错误。

## 状态栏需求

状态栏应显示：

- 当前文件名；未打开文件时显示空状态。
- 侧边栏当前目录或选中文件。
- 不常驻显示 key hints；帮助信息只在 `h` 弹窗中显示。
- 当前视图模式。
- 当前行范围和总行数。
- 当前页码。
- 总列数。
- 当前筛选条件；筛选条件过长时截断显示。
- 最近一次错误或警告。

如果总行数计算代价过高，状态栏可以显示未知、计算中或统计失败状态。

## 验收标准

- 能说明第一阶段做什么、不做什么。
- 后续实现不引入非目标功能，除非先更新本文档。
- 快捷键行为与 Python 原型保持可解释的一致性。

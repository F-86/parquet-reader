# 02. CLI 与错误处理

## 实现目标

本阶段实现命令行入口、启动校验、统一错误类型和用户可见错误信息。完成后，即使 TUI 和数据读取还未完整实现，程序也应该能清晰处理启动失败路径，并支持不带文件路径进入 TUI 空状态。

## CLI 行为

- 接收可选的单个 Parquet 文件路径参数。
- 参数缺失时不报错，进入 TUI 空状态。
- 用户可在 TUI 空状态通过左侧文件列表选择文件，`d` 用于聚焦/切换文件列表。
- 如果启动时传入文件路径，文件不存在时返回非零状态码，并显示路径相关错误。
- 如果启动时传入文件路径，文件不可读时返回非零状态码，并显示权限或读取错误。
- 参数解析层只负责启动配置，不做数据查询。


## 技术选型

### CLI 参数解析

| 候选库 | 优点 | 代价 | 结论 |
|--------|------|------|------|
| `clap` | derive 模式成熟；自动生成 help；错误提示、默认值、枚举参数和后续子命令扩展都较完整 | 依赖和编译成本高于极简解析器 | **推荐**。本项目是面向用户的 CLI/TUI 工具，清晰 help 和错误提示比极小依赖更重要 |
| `pico-args` | 小而直接；适合参数很少的工具 | 需要自己组织 help、错误信息和后续扩展 | 暂不选。除非后续明确追求极小二进制 |
| `argh` | derive 体验简单；依赖相对轻 | 生态和扩展能力弱于 `clap` | 暂不选。适合更简单的内部工具 |

第一阶段建议使用 `clap`，只定义单文件路径、页大小等少量参数。不要在 CLI 层读取 Parquet 内容。

### 错误处理

| 候选库 | 优点 | 代价 | 结论 |
|--------|------|------|------|
| `thiserror` | 适合定义结构化错误枚举；不把具体错误展示策略强加给调用方 | 需要自己设计错误类型 | **推荐**。本项目需要区分启动、初始化、查询、渲染错误 |
| `anyhow` | 适合快速原型和应用顶层错误传播 | 类型边界弱，不适合作为核心层错误接口 | 可在二进制入口少量使用，但不作为模块间公共错误类型 |
| `color-eyre` | 调试报告友好，适合开发期 panic/error 报告 | 输出较重，TUI 用户可见错误不应默认这么详细 | 可选开发辅助，不作为第一阶段必需依赖 |

第一阶段建议：核心模块使用 `thiserror` 定义 `AppError`；二进制入口可以直接把错误格式化为短消息。

## 建议配置类型

```rust
pub struct CliArgs {
    pub file_path: Option<PathBuf>,
}

pub struct AppConfig {
    pub initial_file_path: Option<PathBuf>,
    pub page_size: usize,
    pub root_directory: PathBuf,
}
```

## 错误分类

| 错误类型 | 示例 | 处理方式 |
|----------|------|----------|
| 启动错误 | 传入路径不存在、不可读 | 退出并返回非零状态码 |
| 初始化错误 | Schema 或 metadata 读取失败 | 明确提示；可选择不进入 TUI |
| 查询错误 | 筛选语法错误、count 失败 | 保留 TUI，上报状态栏，不清空已有上下文 |
| 渲染错误 | 终端尺寸过小、绘制失败 | 尽量降级；无法恢复时退出并恢复终端状态 |

## 用户可见错误原则

- 错误信息要短、明确、可行动。
- 启动错误可以直接输出到 stderr。
- TUI 内错误优先显示在状态栏或临时消息区域。
- 内部错误链可以保留给调试日志，但不要默认输出完整数据行。

## 终端恢复原则

TUI 相关代码必须统一管理终端生命周期：

- 进入 alternate screen。
- 启用 raw mode。
- 退出时恢复 raw mode 和 screen。
- panic 或错误路径也尽量恢复终端状态。

本阶段可以先定义接口或 guard 类型，实际 TUI 阶段再补完整实现。

## 验收标准

- 不传文件路径时能进入 TUI 空状态。
- 启动时传入的文件路径不存在时有明确错误。
- CLI 代码不读取 Parquet 内容。
- TUI 内文件选择失败时不退出程序。
- `root_directory` 必须是程序启动时的当前工作目录，侧边栏不能浏览到该目录之外。
- 错误类型能表达启动、初始化、查询和渲染错误。
- 只实现 CLI 时也能通过 `cargo check`。

# 08. 测试策略

## 实现目标

本阶段补齐测试，确保已实现功能在状态、数据访问、格式化和 TUI 行为上可验证。


## 测试库选型

| 测试领域 | 推荐库 | 候选方案 | 选择理由 |
|----------|--------|----------|----------|
| CLI 集成测试 | `assert_cmd` | 手写 `std::process::Command` | 简化运行 cargo binary、断言 exit code、stdout、stderr |
| 临时文件和目录 | `tempfile` | 手写临时路径、`temp-file` | 跨平台临时目录和自动清理更稳，适合生成小型 Parquet 样例 |
| 文本/状态栏快照 | `insta`（可选） | 普通 `assert_eq!` | 适合后续稳定状态栏、帮助文本、渲染片段；第一阶段可先不用 |
| 普通单元测试 | Rust 内置 test | 第三方测试框架 | 状态机、格式化、分页边界不需要额外框架 |

依赖引入顺序：

1. 先用 Rust 内置 test 覆盖 App State 和 Formatting。
2. 实现 CLI 后加入 `assert_cmd`。
3. 需要生成临时 Parquet 文件时加入 `tempfile`。
4. TUI 输出稳定后再考虑 `insta`，避免早期频繁更新快照。

## 单元测试

- 分页边界：第一页、最后一页、空结果、上一页不小于 0。
- 视图切换：Schema/Data 互切不破坏筛选条件。
- 筛选提交：offset 重置，状态栏包含筛选状态。
- 状态转移：每个 Action 的前置条件和状态变化。
- 值格式化：NULL、长字符串、宽字符、二进制、嵌套值。
- 状态栏：有无总数、有无筛选、有无错误。

## 集成测试

- 用小型临时 Parquet 文件验证 Schema、count、page。
- 用空 Parquet 文件验证空数据展示。
- 用多列宽表验证水平滚动状态。
- 用非法筛选表达式验证错误可恢复。
- 用复杂类型字段验证显示降级策略。

## 手动验收

- 打开含 100 行以上的文件，验证翻页。
- 打开宽表，验证水平滚动和最左/最右跳转。
- 输入有效筛选，验证总数和页码刷新。
- 输入无效筛选，验证程序不崩溃。
- 切换 Schema 视图，再切回数据视图，验证状态保留。
- 退出后终端状态恢复正常。

## 测试数据原则

- 不提交真实业务数据。
- 不提交隐私数据。
- 不提交大型样例 Parquet 文件。
- 小型测试样例应可自动生成，优先使用临时目录。

## 风险回归清单

| 风险 | 回归测试 |
|------|----------|
| Count 代价过高 | Count 可失败或未知，第一页仍可展示 |
| 复杂类型显示不可控 | 每类复杂类型至少有格式化测试 |
| 宽字符截断错误 | 中文和 emoji 截断测试 |
| 筛选表达式语义不清 | 无效筛选错误展示测试 |
| 终端恢复失败 | 手动退出和错误退出验收 |

## 验收标准

- 状态机核心动作有单元测试。
- 格式化边界有单元测试。
- 数据访问至少有小文件集成测试。
- 手动验收清单可以逐项执行。
